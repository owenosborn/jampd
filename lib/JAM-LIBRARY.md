# Jam Library

The Jam library is a collection of Lua modules designed to play nicely with the Jam system. Each module can be loaded with `require()`.

## chord

Parses chord symbols into pitch arrays. Supports major, minor, diminished, and augmented qualities with extensions (6, 7, 9, 11, 13).

```lua
require("lib/chord")
```

### Chord symbols

Roots: `C`, `D`, `E`, `F`, `G`, `A`, `B` with `#` or `b` accidentals

Qualities:
- (nothing) = major — `C`, `F#`
- `-` = minor — `C-`, `F#-`
- `+` = augmented — `C+`
- `o` = diminished — `Co`

Extensions: `6`, `7`, `maj7`, `9`, `b9`, `7b9`, `7#9`, `11`, `#11`, `13`, `b13`, `7b5`, `sus4`

Comma-separate multiple extensions: `A-7b5` or `C7,b9`

Slash chords for bass: `C/E`, `G7/B`

Examples: `C-7`, `F#maj7`, `Bb7b5`, `Abo7`, `D-7b5`

### Creating chords

```lua
local chord = Chord.new("A-7")
```

### chord:note(index, octave)

Get a specific chord tone as a MIDI note number. Index wraps around, so you can count past the number of tones to walk through the chord across octaves.

```lua
chord:note(1, 5)   -- root at octave 5
chord:note(2, 5)   -- second tone at octave 5
chord:note(5, 5)   -- wraps: same as note(1, 5) for a 4-note chord
```

### chord:filter(note)

Snap any MIDI note number to the nearest chord tone. Useful for constraining melodies or controller input to the current harmony.

```lua
chord:filter(62)              -- snaps to nearest tone of the chord
chord:filter(math.random(40, 80))  -- random note, quantized to chord
```

### chord:play(jam, velocity, duration, octave)

Play all chord tones at once.

```lua
chord:play(jam, 80, 1/2, 5)
```

### chord:voice(center)

Generate a close voicing around a center MIDI note. Returns the chord for chaining. After calling `voice()`, use `playv()` to play the voicing.

```lua
chord:voice(60):playv(jam, 80, 1/2)
```

### chord:playv(jam, velocity, duration)

Play the voicing generated by `voice()`. Must call `voice()` first.

### chord.tones

The raw array of intervals (relative to root). For example, `Chord.new("C-7").tones` is `{0, 3, 7, 10}`.

### chord.root

Root pitch class (0–11). C=0, C#=1, ..., B=11.

## progression

Manages a sequence of chords with timing, driven by the jam tick.

```lua
require("lib/progression")
```

### Creating progressions

From a string, where dots set duration (chord name = 1 beat, each additional dot = 1 more beat):

```lua
-- Dm7 for 1 beat, G7 for 1 beat, Cmaj7 for 2 beats
local prog = Progression.new("D-7.G7.Cmaj7...")

-- Bb for 4 beats, F7 for 4 beats
local prog = Progression.new("Bb...F7...")
```

Or build programmatically:

```lua
local prog = Progression.new()
prog:add(Chord.new("C-7"), 4)   -- C minor 7, 4 beats
prog:add(Chord.new("F7"), 4)    -- F7, 4 beats
```

### progression:tick(jam)

Call every tick. Advances the playhead and returns the current chord. Loops automatically.

```lua
function tick(jam)
    local chord = prog:tick(jam)
    -- chord is a Chord object, use chord:note(), chord:filter(), etc.
end
```

### progression:isnew()

Returns true on the tick when the chord changes. Useful for triggering something once per chord.

```lua
if prog:isnew() then
    chord:print()
end
```

### progression:chord()

Get the current chord without advancing.

### progression:reset()

Reset playhead to the beginning.

### progression:scale(factor)

Scale all durations by a factor. `prog:scale(2)` doubles the length.

## sequencer

A loop recorder and player. Records note and knob events with tick-accurate timing, plays them back in a loop. Supports arm/record/play/stop states, beat-synced playback, and preset transitions.

```lua
local Sequencer = require("lib/sequencer").Sequencer
```

### Creating a sequencer

```lua
local seq = Sequencer.new({
    tpb = jam.tpb,
    max_events = 1000,
    output = function(type, ...)
        if type == "note" then
            local note, vel, dur = ...
            jam.noteout(note, vel, dur)
        end
    end
})
```

### Recording

```lua
seq:arm()                        -- wait for first note to start recording
seq:recordNote(note, velocity)   -- call from notein
seq:endRecording()               -- stop and quantize loop length to nearest beat
```

### Playback

```lua
seq:play()       -- start immediately
seq:playSync()   -- start on next beat boundary
seq:stop()       -- stop and send note-offs for held notes
seq:toggle()     -- toggle play/stop
seq:tick()       -- call every tick
```

### State queries

```lua
seq:isRecording()
seq:isPlaying()
seq:isArmed()
seq:isStopped()
seq:hasEvents()
seq:getState()     -- returns "ARMED", "RECORDING", "PLAYING", "STOPPED", or "SYNCING"
```

### Saving and loading

```lua
local data = seq:serialize()       -- returns table of events + loop length
seq:deserialize(data)              -- load saved sequence
seq:deserialize(data, true)        -- load immediately (even if playing)
```

If the sequencer is playing when you call `deserialize`, the new sequence is staged and swapped in seamlessly at the next loop boundary.

## subjam

Run jam scripts inside other jam scripts. Each sub-jam gets its own namespace, so variables don't collide. Output can be routed through a callback for chaining jams together.

```lua
local SubJam = require("lib/subjam")
```

### Loading a sub-jam

```lua
local drums = SubJam.load("path/to/drums.lua", jam)
```

### Ticking

```lua
function tick(jam)
    drums.tick(jam)
end
```

### Routing input

```lua
if drums.notein then
    drums.notein(note, velocity)
end
```

### Output routing

By default, a sub-jam's `noteout` and `msgout` go to the parent jam. Pass a callback to intercept and reroute:

```lua
local processor = SubJam.load("generator.lua", jam, function(type, ...)
    if type == "note" then
        local note, vel, dur = ...
        -- route to another sub-jam, transform, filter, etc.
        other_jam.notein(note, vel)
    end
end)
```

This lets you build pipelines: one jam generates notes, another arpeggiates them, another adds effects.

## latch

Captures and holds notes even after keys are released — like a sustain pedal that resets when you play new notes.

```lua
local Latch = require("lib/latch").Latch
```

### Usage

```lua
local latch = Latch.new(function(note, velocity)
    jam.noteout(note, velocity)
end)

latch:enable()

function notein(jam, note, velocity)
    latch:notein(note, velocity)
end
```

When enabled, notes stay sounding after release. Playing a new note (or chord) after fully releasing all keys clears the previous latched notes first.

### Methods

```lua
latch:enable()
latch:disable()     -- sends note-offs for all latched notes
latch:toggle()
latch:get_notes()   -- array of latched note numbers in press order
latch:clear()       -- reset without sending note-offs
```

### Saving and recalling

```lua
local saved = latch:get_notes_with_velocity()  -- array of {note, velocity}
latch:recall(saved)                             -- restore from saved state
```

## presets

File-based preset management. Saves and loads Lua tables to numbered files in a directory.

```lua
local Presets = require("lib/presets").Presets
```

### Usage

```lua
local presets = Presets.new("path/to/presets")

-- Save
presets:save({ bpm = 120, scale = {0, 2, 4, 5, 7} })

-- Navigate
local settings = presets:next()
local settings = presets:prev()
local settings = presets:load(3)

-- Info
presets:count()              -- number of presets
presets:getCurrentName()     -- e.g. "0003"
presets:getDisplayString()   -- e.g. "3 of 7"

-- Delete current
presets:delete()
```

## notate

Captures notes played by a jam and writes them to a LilyPond file for music notation.

```lua
require("lib/notate")
```

### Usage

```lua
function init(jam)
    notate = Notate.new(jam, "/path/to/output.ly")
end

function tick(jam)
    -- play notes as usual — notate captures them transparently
    if jam.every(1) then
        jam.noteout(60, 100, 1)
    end

    -- after enough beats, write the file
    if jam.once(32) then
        notate:finish()    -- write the .ly file
        notate:restore()   -- stop capturing, restore original noteout
    end
end
```

Compile the output with `lilypond output.ly` to get sheet music as PDF.

## utils

General-purpose helpers. All functions are global after requiring.

```lua
require("lib/utils")
```

### Probability

```lua
p(0.5)       -- 50% chance of true (alias: prob())
```

### Random numbers

```lua
randi(1, 10)    -- random integer 1 to 10
randi(5)        -- random integer 0 to 5
randf(0.5, 2.0) -- random float 0.5 to 2.0
randf()          -- random float 0.0 to 1.0
```

### Choice

```lua
choose({60, 64, 67})              -- random element from array
weighted_choose({10, 1, 1})       -- weighted random, returns index
```

### Math

```lua
clamp(value, 0, 127)              -- constrain to range
lerp(a, b, 0.5)                   -- linear interpolation
map(value, 0, 127, 0.0, 1.0)      -- remap from one range to another
wrap(value, 0, 12)                -- wrap to range [0, 12)
```

### Counter

A simple wrapping counter.

```lua
local c = Counter.new(4)   -- counts 0, 1, 2, 3, 0, 1, ...
c:tick()                    -- advance and return current value
c:reset()                   -- back to 0
```
